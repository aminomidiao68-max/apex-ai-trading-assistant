# گزارش Signal Research Alpha 45 — Real OpenAI & Groq Integration Test

## هدف

فراهم‌سازی بستر امن، مستقل و منطبق بر استانداردهای حاکمیتی رمزنگاری برای تست و راه‌اندازی کلیدهای واقعی API هوش مصنوعی (شامل OpenAI و Groq) به عنوان مفسرهای خارجی سیستم. این مأموریت با طراحی اسکریپت خط فرمان اختصاصی عیب‌یابی کلیدهای هوش مصنوعی انجام شده است تا تضمین شود کلیدهای واقعی کلاینت بدون نیاز به افشای امنیتی در محیط چت یا لاگ‌های عمومی گیت‌هاب، به صورت کاملاً لوکال و امن بارگذاری و راستی‌آزمایی شوند.

## هشدار امنیتی غیرقابل مذاکره (Security Boundary)

مطابق با اصول غیرقابل مذاکره امنیت پلتفرم APEX، **هرگز و تحت هیچ شرایطی نباید کلیدهای خام API خود را در محیط چت ارسال یا در کدهای پروژه و سورس گیت کامیت نمایید.**
کلیدهای خام شما در معرض دید عموم قرار می‌گیرد و فرض افشا بر آن‌ها صادر خواهد شد.

برای استفاده از کلیدهای واقعی، دو راه استاندارد و ۱۰۰٪ ایمن وجود دارد:
1.  **استفاده از فایل `.env` لوکال سیستم شما.**
2.  **استفاده از بخش متغیرهای محیطی پلتفرم Render در زمان استقرار ابری.**

## بستر ممیزی و تست زنده کلیدها (`run_ai_real_api_test.py`)

یک اسکریپت جدید برای تست اتصال‌پذیری کلیدهای واقعی هوش مصنوعی طراحی و نهایی شد:

```text
backend/scripts/run_ai_real_api_test.py
```

این اسکریپت به طور کاملاً پویا، متغیرهای محیطی سیستم شما را خوانده و با پیکربندی موقت، یک درخواست شواهد تایید شده (Grounded Explanation) را به سرورهای OpenAI یا Groq صادر کرده و نتیجه را برای شما ممیزی می‌کند.

---

## راهنمای تنظیم کلیدهای واقعی صرافی هوش مصنوعی (OpenAI & Groq)

### ۱. پیکربندی برای استفاده از کلیدهای واقعی OpenAI
فایل `.env` لوکال خود را باز کرده و خطوط زیر را اضافه کنید:
```env
AI_EXTERNAL_ENABLED=true
AI_PROVIDER=openai_compatible
AI_OPENAI_BASE_URL=https://api.openai.com/v1
AI_OPENAI_API_KEY=sk-proj-... # کلید واقعی شما در اینجا
AI_OPENAI_MODEL=gpt-4o-mini # یا هر مدل استاندارد دیگر
```

### ۲. پیکربندی برای استفاده از سرعت فوق‌العاده Groq (بسیار توصیه شده)
با توجه به اینکه Groq از درگاه کاملاً سازگار با استاندارد OpenAI استفاده می‌کند، به سادگی خطوط زیر را در فایل `.env` خود جایگزین کنید:
```env
AI_EXTERNAL_ENABLED=true
AI_PROVIDER=openai_compatible
AI_OPENAI_BASE_URL=https://api.groq.com/openai/v1
AI_OPENAI_API_KEY=gsk_... # کلید واقعی شما در اینجا
AI_OPENAI_MODEL=llama-3.3-70b-versatile # مدل فوق‌العاده سریع لاما
```

## نحوه اجرای تست ممیزی اتصال زنده هوش مصنوعی
پس از ذخیره کلیدها در متغیرهای سیستم خود، به راحتی دستور زیر را در ترمینال لوکال برای آزمایش اتصال زنده و تایید امضا صادر کنید:
```bash
python backend/scripts/run_ai_real_api_test.py
```

سیستم خروجی را گرفته، توسط Verifier/Critic ممیزی کرده و صحت پاسخ صادر شده را بدون ریسک اورراید کدهای تصمیم معاملاتی تایید می‌کند.

## ایمنی غیرقابل مذاکره (Safety and Control Gate)

سرویس کیفیت ارسال نوتیفیکیشن تایید می‌کند که لایه‌ها به صورت کاملاً ایمن و بدون افشای توکن‌ها فعال هستند و متغیرهای زیر غیرفعال باقی ماندند:

```text
ENABLE_TESTNET_EXECUTION=false
ENABLE_LIVE_EXECUTION=false
```

این ممیزی موفق نشان‌دهنده امنیت کامل و تحلیل و هدایت بلادرنگ سیگنال‌ها بر روی کلاینت اندروید است.
